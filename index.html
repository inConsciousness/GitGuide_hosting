Comprehensive Git Guide

Basic Git Workflow

1. Initialize a Repository

git init

Initialize a new Git repository in the current directory.

2. Clone an Existing Repository

git clone <repo-url>

Clone a repository into a new directory.

3. Check Repository Status

git status

Shows the current state of the working directory and staging area.

4. Add Files to Staging Area

git add <file-name>

Add files to the staging area before committing.

5. Commit Changes

git commit -m "Commit message"

Commit changes with a descriptive message.

6. Push Changes to Remote

git push origin <branch-name>

Push changes to a remote repository.

7. Pull Latest Changes

git pull origin <branch-name>

Fetch and integrate changes from a remote repository.

Branch Management

1. Create a New Branch

git branch <branch-name>

Create a new branch.

2. Switch Branches

git checkout <branch-name>

Switch to a different branch.

3. Merge Branches

git merge <branch-name>

Merge changes from one branch into the current branch.

4. Delete a Branch

git branch -d <branch-name>

Delete a branch.

Git Collaboration

1. Fork a Repository

Fork a repository on GitHub to make your own copy.

2. Create a Pull Request

Submit a pull request to propose changes to the original repository.

Git Flow

1. Main Branch

The production-ready branch.

2. Develop Branch

The branch where features and fixes are integrated before being pushed to `main`.

3. Feature Branch

A branch for developing new features.

4. Release Branch

A branch for preparing new production releases.

5. Hotfix Branch

A branch for fixing urgent bugs in the production code.

git flow init - Initialize Git Flow
git flow feature start <feature-name> - Start a feature branch
git flow feature finish <feature-name> - Finish a feature branch
git flow release start <version> - Start a release branch
git flow release finish <version> - Finish a release branch
git flow hotfix start <version> - Start a hotfix branch
git flow hotfix finish <version> - Finish a hotfix branch
Git Best Practices
  • Commit Often: Commit your changes frequently, with clear and concise messages.
  • Use Branches: Keep your `main` branch clean by using feature branches for development.
  • Keep a Clean History: Rebase and squash commits to maintain a clean commit history.
  • Write Descriptive Messages: Your commit messages should describe the “why” behind the change, not just the “what.”
